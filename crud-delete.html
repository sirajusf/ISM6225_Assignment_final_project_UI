Delete or Archive Records

This page explains how incorrect, duplicate, or policy-violating records could be removed.

How Delete Would Work

Moderators can either hard-delete spam/invalid submissions or archive records that should not be visible publicly but must remain in system history.

  • Soft delete: hides record from dashboard but keeps audit trail
  • Hard delete: removes test or malicious records permanently after review
  • Confirmation step: requires reason code and secondary approval

Example Remove Action

Record: TIP-2026-0388

Reason: Duplicate of TIP-2026-0387 with copied evidence links

Action: Archived and merged with original report
